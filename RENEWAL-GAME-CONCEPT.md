# 카드 가챠 리뉴얼 게임 콘셉트

> 기획 저장용 문서. 현재 기능 구현 대상 아님.

> 운영 원칙: 사용자가 별도로 지시하기 전까지 로컬 작업과 검증만 진행한다. 커밋, Git push와 배포를 하지 않는다.

## 전체 구조

- 기존 카드뽑기와 SOOP 후원 연동은 유지한다.
- 게임 콘텐츠를 다음 두 종류로 구분한다.
  - 미니게임: 카드뽑기 포인트 또는 미니게임 한정판 카드를 획득한다.
  - 메인게임: 보유한 카드를 활용해 다른 유저와 플레이한다.

## 개발 우선순위

- 첫 작업은 기존 전체 카드 명단을 확인하고 모든 카드를 새 `F~SSS` 등급으로 재배치하는 것이다.
- 카드별 등급표를 먼저 확정한 뒤 등급 확률, 능력치, 강화와 게임 콘텐츠를 설계한다.
- 현재 개발 목표는 공통 카드·강화 시스템, 미니게임 2종, 경매장과 `정식게임 2: 1대1 카드 듀얼`이다.
- `정식게임 1: 2인 협동 랜덤 카드 디펜스`는 현재 개발 범위에서 제외한다.
- 협동 디펜스는 실시간 2인 동기화, 맵과 경로, 일반 몹·정예 몹·보스 이미지, 웨이브, 이동·전투 연출과 인물별 능력이 모두 필요해 제작량이 가장 크다.
- 협동 디펜스의 실제 구현 여부는 추후 결정하며 현재는 콘셉트만 보관한다.
- 현재 목표 콘텐츠 사이의 정확한 개발 순서는 추후 확정한다.

## 공통 카드 콘셉트

### 등급 체계

- 모든 리뉴얼 콘텐츠에서 공통으로 사용하는 카드 등급은 총 9단계다.
- 낮은 등급부터 `F → E → D → C → B → A → S → SS → SSS` 순서다.
- `EX`는 미니게임 보상과 경매장을 통해 획득할 수 있는 특수 등급이다.
- `EX` 카드는 일반 카드뽑기와 상점에서 획득할 수 없다.
- `EX`는 도감 박제와 수집을 위한 기념 카드이며 전투 등급이 아니다.
- `EX` 카드는 미니게임과 메인게임의 덱 편성, 파견, 강화 및 전투에 참여할 수 없다.
- 현재 카드 212장의 새 등급은 `data/cards.json`의 `renewalRarity` 필드로 배치했다.
- 단체사진 카드 8장은 모두 `EX`로 배치하고 일반 등급 분포에서 제외한다.
- 리뉴얼 등급 분포는 `F 17 / E 14 / D 21 / C 21 / B 33 / A 32 / S 34 / SS 25 / SSS 2 / EX 8`이다.
- 등급별 능력치, 등장 확률, 강화 한도와 시각 효과는 추후 확정한다.

### 인물별 등급 배치 원칙

- 기존 카드 등급 배치에서 사용한 인물 간 우선순위를 새 `F~SSS` 체계에도 유지한다.
- 인물 전체를 하나의 등급으로 고정하지 않고 해당 인물의 여러 카드를 상·중·하위 등급에 분산한다.
- 김윤환 카드는 전체 인물 중 가장 높은 등급 위주로 배치하고 대표 카드를 SSS에 둔다.
- 토마토는 김윤환 다음 순위의 상위등급 위주로 배치한다.
- 지두두는 토마토 다음 순위로 배치한다.
- 나머지 여자 인물 카드는 전반적으로 중상위 등급 비중을 높인다.
- 김민철과 변현제는 일반 남자 인물보다 높게, 여자 인물들과 비슷한 중상위권으로 배치한다.
- 나머지 남자 인물은 상대적으로 하위등급 비중을 높이되 지나치게 낮은 등급에만 몰지 않는다.
- 남자 인물도 인물별로 최소 한 장은 중상위 카드가 나오도록 배치한다.
- 기존 카드별 상대 우선순위와 인물별 배치 원칙을 함께 반영했으며 `scripts/assign-renewal-rarities.js`로 검증하고 재현할 수 있다.

### 카드 강화 시스템

- 강화 재료에는 해당 등급 카드의 최초 1장을 제외한 중복 보유분만 사용할 수 있다.
- 기본 재료 수량은 강화 시도 1회당 카드 3장이다.
- 최대 강화 단계는 `+9`다.
- `EX` 카드는 강화할 수 없다.
- 강화 수치는 시즌2 모험과 월드보스 전투력에 적용한다. 2인 협동 랜덤 카드 디펜스가 확정되면 같은 카드 전투력 규칙을 재사용한다.
- 1대1 카드 듀얼에서는 카드의 강화 단계와 강화 보너스를 적용하지 않는다.
- 강화 성공과 실패 결과는 서버에서 판정하고 모든 확률을 강화 화면에 공개한다.

| 강화 대상 등급 | 사용 가능한 중복 재료 |
|---|---|
| F | F 등급 1장 |
| E | F 등급 3장 |
| D | E 등급 3장 |
| C | D 등급 3장 |
| B | C 등급 3장 |
| A | B 등급 3장 |
| S | A 등급 3장 |
| SS | S 등급 3장 |
| SSS | SS 등급 3장 또는 SSS 등급 1장 |

- SSS 강화에서는 `SS 중복 3장`과 `SSS 중복 1장` 중 사용할 재료 방식을 유저가 선택할 수 있다.
- 한 번의 강화 시도에서 두 재료 방식을 섞어 사용할 수 없다.

강화 성공률은 `단계별 기본 성공률 - 카드 등급 페널티`로 계산한다. 등급 페널티는 퍼센트가 아니라 퍼센트포인트로 차감한다.

| 목표 단계 | 기본 성공률 | 파괴율 | 실패 결과 |
|---|---:|---:|---|
| +1 | 100% | 0% | 없음 |
| +2 | 100% | 0% | 없음 |
| +3 | 100% | 0% | 없음 |
| +4 | 95% | 0% | 단계 유지, 재료 소모 |
| +5 | 90% | 0% | 단계 유지, 재료 소모 |
| +6 | 85% | 0% | 단계 유지, 재료 소모 |
| +7 | 75% | 2% | 일반 실패는 단계 유지, 파괴 시 본 카드와 재료 소모 |
| +8 | 65% | 5% | 일반 실패는 단계 유지, 파괴 시 본 카드와 재료 소모 |
| +9 | 55% | 10% | 일반 실패는 단계 유지, 파괴 시 본 카드와 재료 소모 |

| 카드 등급 | 성공률 페널티 |
|---|---:|
| F | 0%p |
| E | -2%p |
| D | -4%p |
| C | -6%p |
| B | -8%p |
| A | -10%p |
| S | -12%p |
| SS | -15%p |
| SSS | -18%p |

- 일반 실패율은 `100% - 최종 성공률 - 파괴율`이다.
- `+1~+3`은 카드 등급과 관계없이 항상 100% 성공한다.
- 예시: SSS 카드의 `+9` 성공률은 `55% - 18%p = 37%`, 일반 실패율은 53%, 파괴율은 10%다.

- `+9` 시도에는 위 등급별 중복 카드 재료와 별도의 카드뽑기 포인트가 함께 필요하다.
- `+9` 시도에 사용한 카드뽑기 포인트는 성공, 일반 실패와 파괴 결과 모두에서 소모된다.
- 파괴 결과가 나오면 사용 재료와 아이템·포인트는 소모되지만 원본 카드는 유지한다. 해당 카드의 강화 단계와 카드 EXP만 `0`으로 초기화한다.
- `+7~+9` 시도 전에는 파괴 가능성과 소모 대상을 명확하게 보여주고 별도 확인을 받는다.
- `+9` 포인트 비용, 연속 실패 보정과 파괴 복구 수단은 추후 확정한다.

### 게임별 카드 설계 원칙

- 카드의 인물, 이미지, 소유 여부와 공통 등급은 모든 콘텐츠에서 공유한다.
- 카드의 전투 능력치, 역할, 스킬과 성장 방식은 각 게임의 콘셉트에 맞게 별도로 설계한다.
- 하나의 범용 공격력이나 범용 스킬을 모든 게임에 그대로 적용하지 않는다.
- 같은 카드라도 레이드, 던전, 협동 디펜스와 1대1 듀얼에서 서로 다른 역할을 가질 수 있다.
- 강화 수치는 미니게임 공격력과 2인 협동 랜덤 카드 디펜스에 적용한다.
- 1대1 카드 듀얼은 모든 유저가 동일 카드의 동일한 기본 능력치를 사용한다.

### 미니게임 공통 공격력

- 미니게임에 참여할 수 있는 `F~SSS` 카드는 공통 공격력을 가진다.
- 미니게임 공격력은 카드 등급과 카드 강화 수치에 따라 결정한다.
- 거대 악플러 레이드의 피해량과 방치형 던전의 덱 총합 공격력은 이 값을 사용한다.
- 미니게임에는 지원형, 역할, 고유 스킬, 시너지와 수동 덱 편성 개념을 넣지 않는다.
- 유저가 참가하면 서버가 해당 유저의 보유 카드 중 공격력이 가장 높은 카드 5장을 자동 배치한다.
- 자동 배치된 5장의 공격력 합계를 해당 유저의 미니게임 덱 총합 공격력으로 사용한다.
- `EX` 카드는 게임에 참여할 수 없으므로 공격력을 가지지 않는다.
- 등급별 기본 공격력과 강화 계산식은 카드 강화 기획 확정 후 결정한다.

예시:

- 거대 악플러 레이드: 자동 편성된 카드 5장의 공격력으로 보스 피해 계산
- 방치형 던전: 자동 편성된 카드 5장의 총합 공격력으로 입장 조건과 보상 단계 계산
- 2인 협동 랜덤 카드 디펜스: 소환 비용, 공격 방식, 사거리, 조합 단계, 협동 스킬
- 1대1 카드 듀얼: 사용 비용, 공격력, 체력, 종족 시너지, 카드 효과

## 미니게임 1: 거대 악플러 토벌 레이드

### 기본 콘셉트

- 매일 저녁 서버 메인 화면에 체력 1억의 `거대 악플러` 보스가 출현한다.
- 서버 전체 유저가 함께 보스를 공격하는 실시간 공동 이벤트다.
- 유저가 참가를 누르면 보유 카드 중 공격력이 가장 높은 카드 5장이 자동 배치된다.
- 자동 배치된 카드는 보스를 자동 공격한다.
- 카드 공격력은 카드 등급과 카드 강화 수치로 결정한다.
- 카드 강화 시스템의 방식과 규칙은 추후 사용자 설명을 받은 뒤 확정한다.

### 핵심 재미

- 모든 참가자의 실시간 누적 피해량을 딜미터기와 랭킹 보드에 표시한다.
- 높은 순위를 위해 더 좋은 카드를 획득하고 기존 카드를 강화하게 만든다.
- 카드 수집, 중복 카드 활용, 강화, 서버 경쟁을 하나의 순환 구조로 연결한다.

### 구현 시 지켜야 할 방향

- 피해량은 서버에서 계산한다. 클라이언트가 전달한 피해량을 신뢰하지 않는다.
- 유저별 피해를 매초 DB에 기록하지 않는다. 서버가 DPS와 참전 시간을 기준으로 누적 피해를 계산하고 일정 간격으로 집계한다.
- 참전 후 브라우저를 닫아도 자동 공격이 이어지는 구조를 우선 검토한다.
- 레이드 참가 시 자동 편성된 카드와 강화 수치는 해당 레이드 동안 고정한다.
- 고정 체력 1억은 초기 콘셉트다. 실제 운영에서는 참가자 수와 전체 화력에 따른 조절 기능을 검토한다.

### 보상 방향

- 참여 보상
- 개인 누적 피해 구간 보상
- 보스 처치 시 전체 참가자 보상
- 개인 딜 랭킹 보상
- 카드뽑기 포인트 또는 레이드 한정판 카드

세부 보상량, 레이드 시간과 랭킹 보상은 추후 확정한다. 카드 강화 규칙은 별도 설명 전까지 설계하지 않는다.

## 미니게임 2: 방치형 던전 탐험대

### 기본 콘셉트

- 참가 시 자동 편성된 공격력 상위 카드 5장을 던전에 파견하는 방치형 파밍 RPG다.
- 유저가 전투를 직접 조작하지 않는다.
- 카드를 파견하면 30분 뒤 탐험을 마치고 보상을 가져온다.

### 던전 구조

- 초급 던전
- 중급 던전
- 지옥 던전
- 상위 던전일수록 더 높은 덱 능력치를 요구하고 더 좋은 보상을 제공한다.
- 지옥 던전은 `덱 총합 공격력 10,000 이상`만 입장할 수 있다.

### 핵심 보상

- 카드뽑기 포인트
- 상점에서는 구매할 수 없는 `전설의 카드팩`
- `특수 스킬 부여권`
- 던전 등급에 따라 보상의 종류와 품질이 상승한다.

### 핵심 재미

- 카드 수집과 강화로 자동 편성 덱의 총합 공격력을 높인다.
- 덱 능력치가 오르면 더 높은 던전이 해금된다.
- 높은 던전에서만 얻을 수 있는 희귀 보상을 반복 파밍한다.
- 카드 성장, 던전 해금, 희귀 보상 획득이 이어지는 장기 성장 구조를 만든다.

파견 중 다른 콘텐츠 사용 가능 여부, 던전별 입장 조건, 성공 확률과 보상 확률은 추후 확정한다.

## 경매장: 정시 비공개 카드 입찰

### 기본 콘셉트

- 유저가 카드를 직접 등록하는 거래소가 아니라 시스템이 카드를 출품하는 경매장이다.
- 매일 한국시간 기준 `20:00`, `21:00`, `22:00`에 특정 카드가 한 번씩 출품된다.
- 각 경매는 정각부터 30분 동안 비공개 입찰을 받는다.
- 입찰 마감 후 다음 정각 전까지 29분 동안 최종 낙찰금액만 공개한다.
- 공개 경매 화면에는 낙찰자의 닉네임, 계정과 식별 정보를 표시하지 않는다.
- 입찰 진행 중에는 다른 유저의 입찰 여부와 입찰 금액을 공개하지 않는다.
- 경매 종료 시점에 가장 높은 금액을 제출한 유저가 자신의 입찰 금액으로 낙찰받는다.
- EX 카드는 아주 드문 확률로 경매장에 출품된다.
- 경매장 EX 카드는 도감 박제용이며 게임 편성, 강화와 전투에는 사용할 수 없다.

### 서버 처리 원칙

- 출품 시작, 입찰 마감과 낙찰 판정은 모두 서버 시간을 기준으로 처리한다.
- 입찰 금액과 최고 입찰자는 마감 전까지 서버에서도 권한 없는 사용자에게 노출하지 않는다.
- 입찰 시 해당 금액을 사용 불가능한 상태로 예약하고, 낙찰 실패 시 반환하는 방식을 우선 적용한다.
- 낙찰 판정과 카드 지급, 낙찰금 차감, 탈락자 예약금 반환은 하나의 원자적 서버 처리로 실행한다.
- 같은 경매의 중복 낙찰과 중복 환불을 막기 위해 경매별 한 번만 마감되는 구조로 만든다.

### 추후 확정 항목

- 한 번에 출품하는 카드 수
- 입찰에 사용하는 재화
- 최저 입찰가와 입찰 단위
- 입찰 금액 수정과 취소 허용 여부
- 최고 입찰가가 같은 경우의 낙찰 기준
- 일반 등급별 출품 확률과 EX 출품 확률
- 유저별 일일 입찰 횟수 제한

## 보류 기획: 2인 협동 랜덤 카드 디펜스

> 구현 여부 미정. 현재 개발 범위에 포함하지 않고 콘셉트만 보관한다.

### 기본 콘셉트

- 두 명의 유저가 한 팀으로 진행하는 실시간 협동게임이다.
- `운빨존많겜`의 랜덤 소환과 조합 성장 구조를 참고한다.
- 각 유저가 카드 가챠에서 실제로 보유한 카드가 게임 내 소환 후보가 된다.
- 게임 중 자신의 보유 카드 풀에서 유닛 카드를 랜덤으로 뽑는다.
- 뽑은 카드를 조합해 더 높은 단계의 유닛으로 성장시킨다.
- 기존 인물 카드의 형태와 수집 가치를 유지하면서 협동 디펜스 게임에 맞게 표현한다.

### 핵심 흐름

1. 두 유저가 각자의 보유 카드로 출전 카드 풀을 구성한다.
2. 게임 재화를 사용해 카드 유닛을 랜덤 소환한다.
3. 소환 결과에 맞춰 같은 카드 또는 정해진 조합을 합성한다.
4. 상위 유닛을 만들어 두 유저가 함께 적의 공세를 막는다.
5. 제한 웨이브 또는 최종 보스를 함께 돌파하면 승리한다.

### 핵심 재미

- 카드 가챠에서 모은 카드가 실제 협동게임의 유닛 풀이 된다.
- 랜덤 소환 결과에 대응해 즉석에서 조합과 배치를 판단한다.
- 두 유저의 카드 풀과 역할이 섞이며 매 경기 다른 전개가 만들어진다.
- 희귀 카드 보유, 카드 성장, 조합 운, 협동 판단을 함께 활용한다.

### 캐릭터 능력 설계 방향

- `운빨존많겜`처럼 게임 밖의 캐릭터 등급과 강화 효과, 게임 안의 캐릭터별 역할과 스킬을 함께 사용한다.
- 기존 게임의 캐릭터, 명칭, 수치와 스킬을 그대로 복제하지 않고 우리 인물 카드에 맞는 능력을 새로 만든다.
- 카드 이미지마다 완전히 다른 스킬을 만들지 않고 `인물별 기본 능력 세트`를 만든다.
- 같은 인물의 여러 카드와 등급은 기본 역할을 공유하고 능력치, 발동 확률, 효과 범위와 추가 효과가 달라진다.
- 인물별 역할 후보는 단일 공격, 범위 공격, 보스 공격, 군중 제어, 강화 지원, 재화 지원과 확률형 특수 능력으로 구분한다.
- 전체 인물 수와 종족 분포를 기준으로 역할 수를 나누고 특정 인물이나 종족에 필수 능력이 몰리지 않게 한다.
- 세부 인물별 능력은 전체 카드 명단과 등급 재배치를 확정한 뒤 설계한다.

### 보유 카드와 인게임 승급

- 게임 중 유닛을 조합해 다음 등급으로 승급하려면 유저가 해당 상위 등급 카드를 실제로 보유해야 한다.
- 보유하지 않은 등급과 카드는 게임 안에서도 소환하거나 승급 결과로 획득할 수 없다.
- 유저가 보유한 가장 높은 카드 등급이 해당 유저의 인게임 승급 상한이 된다.
- 승급 가능한 상위 카드가 없으면 승급 기능을 비활성화하고 인게임 재료를 소비하지 않는다.
- 상위 카드 보유 조건과 인게임 재료 조건을 만족해도 승급은 확정 성공하지 않는다.
- 승급 시도에는 등급별 성공 확률을 적용해 운 요소를 만든다.
- 승급에 실패하면 합성에 사용한 동일 카드 중 1장만 필드에 남고 나머지는 소멸한다.
- 승급 성공률과 연속 실패 보정 적용 여부는 추후 확정한다.
- 조합에 필요한 카드 수와 승급 결과가 동일 인물인지 무작위 상위 카드인지는 추후 확정한다.

### 필드 배치와 동일 카드 스택

- 맵 공간이 한정적이므로 전투 카드의 가독성을 해칠 정도로 카드 크기를 줄이지 않는다.
- 한 칸에는 동일 카드가 최대 5장까지 겹쳐서 배치된다.
- 겹친 카드는 카드 한 장을 기본 크기로 표시하고 `x2~x5` 수량 표시와 겹침 효과로 장수를 보여준다.
- 같은 카드가 이미 배치된 상태에서 추가 소환되면 빈 칸을 먼저 차지하지 않고 기존 카드 스택에 합쳐진다.
- 한 스택이 5장에 도달한 뒤 같은 카드가 추가되면 다른 빈 칸에 새로운 스택을 만든다.
- 스택의 기본 공격력은 `카드 1장 공격력 × 겹친 장수`로 합산한다.
- 필드 전체에서 동일 카드가 합성 조건 수량에 도달하면 합성을 시도할 수 있다.
- 합성 필요 수량, 합성 시 어느 스택부터 소비할지, 스킬 발동 계산은 추후 확정한다.

### 몹 이동과 패배 조건

- 카드 배치 영역 주위에 몹이 계속 순환하는 하나의 전투 경로를 둔다.
- 등장 포털과 탈출 지점은 만들지 않는다.
- 몹은 경로 끝에서 탈출하지 않고 살아 있는 동안 필드 주위를 계속 돈다.
- 필드에 동시에 존재하는 몹의 수가 제한 수량을 넘으면 두 유저 모두 즉시 게임오버된다.
- 게임오버 기준 수량, 보스의 몹 수 계산 방식과 웨이브별 생성량은 추후 확정한다.
- 이미지 자산은 전투 맵 배경, 일반 몹, 정예 몹, 보스와 전투 이펙트 중심으로 제작한다.

### 적 몹 아트 방향

- 적 몹은 스타크래프트 유닛의 분위기와 역할을 오마주한 SF 몬스터로 제작한다.
- 생체 군단형, 인간 기계병형, 초능력 에너지형의 세 계열을 사용한다.
- 원작 유닛의 이름, 외형, 실루엣과 장비를 그대로 복제하지 않고 우리 게임만의 몬스터로 재해석한다.
- 작은 화면에서도 종류를 구분할 수 있도록 계열별 색, 몸체 형태와 공격 이펙트를 명확하게 나눈다.
- 일반 몹은 단순한 형태, 정예 몹은 강화 장식, 보스는 확실히 큰 크기와 독자 실루엣을 사용한다.

맵 형태, 웨이브, 소환 비용, 조합 공식, 카드별 역할, 협동 방식, 실패 조건과 보상은 추후 확정한다.

## 정식게임 2: 1대1 카드 듀얼

### 기본 콘셉트

- 두 유저가 보유 카드를 이용해 실시간으로 대전하는 1대1 턴제 카드게임이다.
- `하스스톤`의 직관적인 카드 소환, 자원 운용, 필드 전투 구조를 참고한다.
- 카드 가챠에서 수집한 인물 카드를 직접 덱에 편성하고 경기에서 사용한다.
- 상대의 핵심 체력을 먼저 소진한 유저가 승리하는 구조를 우선 후보로 둔다.

### 핵심 방향

- 기존 인물 사진, 카드 프레임, 등급, 종족 정보를 그대로 활용한다.
- 카드마다 전투 능력치와 고유 스킬을 부여한다.
- 카드 등급만으로 승패가 결정되지 않도록 덱 구성, 상성, 자원 운용과 플레이 판단을 중요하게 만든다.
- 저그, 테란, 프로토스 종족별 덱 성격과 시너지를 검토한다.
- 기존 게임의 고유 명칭, 카드 효과, UI와 규칙을 그대로 복제하지 않고 별도의 규칙과 정체성을 만든다.

### 카드 설계 방향

- 1대1 듀얼은 하스스톤처럼 카드 수가 많은 것을 전제로 설계한다.
- 협동 디펜스의 인물별 공통 능력 방식과 달리 듀얼에서는 카드 이미지 하나마다 별도의 비용, 공격력, 체력과 효과를 가질 수 있다.
- 모든 카드를 별도 코드로 만들지 않고 공통 효과 템플릿과 키워드를 조합해 많은 카드를 데이터로 관리한다.
- 같은 인물의 카드라도 등급과 이미지가 다르면 서로 다른 능력과 역할을 가질 수 있다.
- 등급별 능력 예산을 정하되 낮은 등급 카드도 저비용, 연계와 특정 덱에서 사용할 가치를 갖게 한다.
- 카드 강화 단계는 듀얼의 비용, 공격력, 체력과 효과에 영향을 주지 않는다.
- 하스스톤의 턴제 자원 증가, 유닛 소환, 필드 전투와 덱 구성 구조를 참고하되 카드명, 효과 문구, 수치, UI와 고유 규칙은 새로 설계한다.

자원 체계, 승리 조건, 덱 크기, 필드 크기, 종족 특성, 카드 능력치와 스킬은 전체 카드 콘셉트를 먼저 확정한 뒤 설계한다.
