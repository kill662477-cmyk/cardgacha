# PDB-1 무안내 플레이테스트

> 역사적 1차 테스트 기록. 이후 계정 레벨 제거·밸런스 `2026.07.18-random-loot-1`·스키마 v2·강화 성공 FX 개선이 적용됐다. 현재 PDB-1 최종 승인은 신규 상태에서 사용자 무안내 15~20분 재테스트 후 결정한다.

> 일자: 2026-07-17
> 환경: 격리 로컬 저장소, PC `1366x768`, 모바일 가로 `844x390`
> 시작 상태: 8,450P, 행동력 64/120, 기본 편성 5장, 스테이지 1-1

## 결과

- 심각도 1: 0건
- 심각도 2: 0건
- 낮은 심각도 UX: 2건 발견, 2건 수정
- 진행 불가, 상태 손실, 중복 차감과 결과 불일치 없음

## 수행 흐름

1. 편성창에서 기존 카드 1장을 해제하고 미편성 카드 1장을 선택해 5장 편성 저장
2. 자동 진행 OFF/ON 전환 후 스테이지 1-1에서 1-5까지 진행
3. 빠른 전투 1회로 행동력 20 소모, 오프라인 보상 112P 수령
4. 일반 보급팩 1개 구매, 50P 차감과 카드 3장 지급 및 개봉 결과 확인
5. 강화 준비 완료 필터에서 사테 선택, 성공 100%·실패 0%·파괴 0%와 C 중복 3장 소모 확인 후 1성 강화
6. 도감 보유·종족·등급 필터, 카드 상세, 잠금·대표카드 상태 확인
7. 카드 짝맞추기 연습 8쌍 완료, 보상 모드 완료로 행동력 10 소모와 76P 지급 확인
8. 카드백 합계 10 연습 모드에서 170칸 보드, 선택 합계 표시와 중도 종료 확인
9. 월드보스 무료 도전 시작, 실시간 피해 증가와 교전 중 버튼 잠금 확인
10. 전투력 랭킹에서 전체 순위, 상위 백분위, TOP 50 차이와 현재 편성 5장 확인
11. 새로고침 후 스테이지, 편성, 포인트, 행동력, 강화와 미니게임 기록 유지 확인

## 수정한 UX 문제

### 현재 대표카드 버튼

- 문제: `대표카드 설정됨` 상태에서도 버튼을 다시 누를 수 있어 같은 상태 토스트만 출력됨
- 수정: 현재 대표카드는 설정 버튼을 비활성화
- 재검증: 새로고침 후 버튼 문구와 disabled 상태 일치

### 미니게임 결과 잔류

- 문제: 결과 화면에서 보상·연습 모드만 변경하면 이전 모드 결과가 중앙에 남음
- 수정: 모드 변경 시 이전 결과를 초기화하고 선택한 모드의 대기 화면 표시
- 재검증: 연습 결과에서 보상 모드 전환 시 결과 숨김, 보상 시작 화면 표시

## 저장 재검증 값

- 스테이지: `1-5`
- 편성: 김윤환, 김윤환, 토마토, 지두두, 김민철
- 포인트: `8,588P`
- 행동력: `34/120`
- 빠른 전투: `1/3`
- 강화: 사테 `1성`, EXP `0/180`
- 미니게임: 최고 `1,520`, 플레이 `5회`, 일일 보상 잔여 `2,924P`

기능 기준 PDB-1은 통과했다. 실제 신규 사용자가 문구만 보고 같은 순서를 수행하는 인지 검증은 최종 공개 전 사람 1회 테스트로 별도 확인한다.
