# PDB-6 요청 상태·오류·충돌 UX

## 상태기

`src/renewal/request-coordinator.js`가 게임 명령별 상태를 관리한다.

```text
idle -> pending -> success
                -> error
                -> offline
                -> auth
                -> conflict
```

- 같은 작업이 `pending`이면 두 번째 호출은 최초 Promise를 반환한다.
- 버튼은 처리 중 `aria-busy="true"`가 되고 추가 입력을 받지 않는다.
- 네트워크가 없으면 작업을 실행하지 않는다.
- 재시도 가능한 실패는 최초 task와 동일한 명령 봉투·멱등성 키를 재사용한다.
- 버전 충돌은 자동 덮어쓰기하지 않고 최신 상태 재조회로 연결한다.

## 화면

| 상태 | 표시 | 사용자 행동 |
|---|---|---|
| 초기 로딩 | 카드 데이터 동기화 | 완료까지 대기 |
| 오프라인 | 서버 명령 정지 | 연결 후 같은 요청 재시도 |
| 세션 만료 | 로그인 만료 | 재인증 후 상태 확인 |
| revision 충돌 | 다른 기기 변경 | 최신 기록 불러오기 |
| 서버 오류 | 트랜잭션 롤백 | 같은 요청 안전 재시도 |

로컬 시각 검사용으로 `?ui-state=loading`, `network`, `offline`, `auth`, `conflict`, `server`를 지원한다. 운영에서는 서버 응답 코드가 상태를 결정한다.

## 적용 작업

- 편성 저장
- 오프라인·빠른 전투 보상 정산
- 카드팩·지원 보급 구매
- 카드 강화
- 미니게임 시작·완료
- 월드보스 공격·보상

카드팩과 강화는 FX가 끝날 때까지 동일 작업 잠금을 유지한다.
