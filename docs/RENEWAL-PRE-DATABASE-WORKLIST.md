# 계정 연동·데이터베이스 작업 전 체크리스트

> 목표: 로컬 게임 동작과 서버 계약을 먼저 잠근 뒤 Supabase 계정 연동을 시작한다.
> 상태: 사용자가 DB 작업 시작을 승인했다. migration·RPC·RLS는 로컬 작성·검증 중이며 운영 실행은 별도 확인 뒤 진행한다.
> 배포: 사용자 지시 전까지 커밋, push와 배포를 하지 않는다.

## PDB-0: UI·UX·브랜드 완성 게이트

DB 준비보다 먼저 [리뉴얼 UI·UX·브랜드 자산 감사](RENEWAL-VISUAL-UX-AUDIT.md)를 완료한다.

필수 순서:

1. 디자인 기준과 게임명 `카드가챠 시즌2` 표기 잠금
2. 심볼, 워드마크, 축약 마크와 favicon 제작
3. 공통 타이포그래피, 색, 버튼, 패널과 상태 UI 잠금
4. F~SSS 세로형과 EX 가로형 카드 프레임 이미지 재제작
5. 모험, 상점, 강화, 도감, 월드보스, 미니게임과 랭킹 UX 정리
6. 카드팩, 미등록 카드, TOP 3와 미니게임 상태 이미지 제작
7. PC와 모바일 가로 전체 화면 시각 승인

완료 기준:

- 임시 텍스트 로고, 빈 검은 이미지와 placeholder 화면 0건
- F~SSS·EX 카드 프레임 이미지와 공용 오버레이 컴포넌트 승인
- 7개 화면의 PC `1366x768`, 모바일 가로 `844x390` 승인
- 주요 터치 영역 최소 `44x44px`
- 시즌 1 팩 이미지와 신규 SF UI의 시각 충돌 제거
- 로딩, 빈 상태, 실패, 성공과 네트워크 오류 상태 확인
- 이후 무안내 플레이테스트를 최종 시각 상태로 진행

## 현재 기준선

- 카드 212장, 5개 지역 50개 스테이지, 편성, 자동 전투와 오프라인 보상 구현
- 카드 경험치, `9성` 강화, 도감 보너스, 상점과 보급품 구현
- 카드 짝맞추기, 캄몬사과게임, 월드보스와 전투력 랭킹 로컬 구현
- 카드 상세, 카드팩 FX, 등급별 영상·효과음, 강화 성공 4단계 FX와 고강화 강도 차등 구현
- 리뉴얼 자동 테스트 25개 묶음 통과
- 현재 저장과 보상 판정은 로컬 프로토타입이며 공개 서비스 보안 기준이 아님

## PDB-1 무안내 플레이테스트

- 완전 신규 로컬 상태로 시작한다.
- 편성 변경, 모험 진행, 보상 수령, 카드팩 구매, 강화와 미니게임을 도움 없이 수행한다.
- 잘못 누르기 쉬운 버튼, 결과를 이해하기 힘든 수치와 모바일 가로 조작 문제를 기록한다.
- 심각도 1은 진행 불가·데이터 손실, 심각도 2는 주요 기능 오작동으로 정의한다.
- 추가 연출 요구와 취향 수정은 별도 목록으로 분리한다.

## PDB-2 밸런스 잠금

잠글 값:

- F~SSS 기본 스탯, 전투력과 강화 배율
- 스테이지 난도, 행동력, 오프라인 보상과 카드 경험치
- 일반·정예·프리미엄·종족팩 확률과 가격
- 강화 성공·실패·파괴 확률, 재료와 포인트 비용
- 도감 보너스 `전투 100% 상한 / 방치 30%`
- 미니게임별 일일 상한 `5,000P`
- 월드보스 KST 17·18·19·20시 4회 스케줄, 회차별 도전 횟수, 피해 보상과 카드 경험치
- SOOP 후원 반영 예정 비율 `별풍선 1개 = 3P`

산출물:

- 단일 밸런스 설정 파일
- 설정 버전
- 하·중·상위 계정 성장 시뮬레이션 결과
- 변경 금지 항목과 운영 조정 가능 항목 구분

## PDB-3 상태 스키마 v2

필수 도메인:

- 계정 프로필, 행동력과 포인트. 계정 레벨·계정 EXP 필드 없음
- 카드 최초 획득, 보유 수량, 잠금, 경험치와 강화 단계
- 5인 편성과 편성 프리셋
- 인벤토리, 강화 보조제와 경험치 버프
- 지역·스테이지 진행, 오프라인 정산 시각과 빠른 전투
- 미니게임 일일 획득량, 시도와 서버 검증용 입력 로그
- 월드보스 회차, 도전, 피해와 보상 수령
- 전투력 스냅숏과 랭킹 표시 데이터

규칙:

- 카드 ID와 등급 ID는 이관 후 바꾸지 않는다.
- 서버 권위 데이터와 화면 전용 캐시를 구분한다.
- `schemaVersion`과 로컬 데모 상태 변환 함수를 둔다.
- EX 단체사진은 전시 전용이며 편성·전투에서 제외한다.

## PDB-4 저장·서비스 경계

화면이 호출할 게임 서비스 인터페이스를 먼저 만든다.

```text
loadSnapshot
updateFormation
claimAdventureRewards
purchasePack
enhanceCard
startMinigame
finishMinigame
attackWorldBoss
getPowerRanking
```

- 현재 `saveState()` 직접 호출을 서비스로 이동한다.
- 시각은 주입 가능한 clock, 난수는 주입 가능한 RNG로 통일한다.
- `local-game-service`가 기존 로컬 플레이를 유지한다.
- 이후 `supabase-game-service`를 추가해도 화면과 FX를 다시 작성하지 않는 구조가 완료 조건이다.

- 행동력 회복·소모, 버프 시작·종료와 오프라인 정산

- 미니게임 시드, 입력 검증, 일일 상한과 포인트 지급

- 멱등성 키
- 서버 시각과 서버 시드
- 버전 충돌 응답
- 재시도 가능한 오류와 불가능한 오류
