# 카드가챠 시즌2 공통 UI 시스템

> 적용 대상: `index.html`
> 기준 화면: PC `1366x768`, 모바일 가로 `844x390`
> 구현 파일: `styles/renewal/tokens.css`, `styles/renewal/main.css`

## 방향

- 실제 카드가 화면의 주인공이다.
- UI는 다크 SF 방송 장비처럼 조용하고 정밀하게 보인다.
- 장식보다 상태, 수치와 명령의 구분을 우선한다.
- 모서리는 각진 체계를 유지한다.
- 로고의 파랑은 브랜드에, 라임은 행동과 진행에만 사용한다.

## 색 역할

| 토큰 | 용도 |
|---|---|
| `--bg` | 전체 배경 |
| `--panel` | 기본 판넬 |
| `--panel-soft` | 보조 판넬 |
| `--surface-raised` | hover와 한 단계 높은 영역 |
| `--surface-selected` | 선택된 메뉴·항목 |
| `--line` | 일반 구분선 |
| `--line-bright` | 조작 가능 경계 |
| `--text` | 제목과 핵심 값 |
| `--muted` | 본문과 보조 값 |
| `--subtle` | 메타데이터 |
| `--acid` | 주요 행동, 현재 진행과 활성 상태 |
| `--brand-blue` | 로고와 브랜드 식별 전용 |
| `--gold` | 희귀 보상과 랭킹 |
| `--red` | 위험, 실패와 월드보스 공격 |
| `--cyan` | 정보와 보조 진행 |

등급색은 카드 프레임과 등급 표시에만 사용한다.

## 타이포그래피

- `8px`: 모바일·소형 카드의 필수 메타데이터만 허용
- `9px`: 영문 눈썹과 분류 라벨
- `10px`: 버튼 보조문과 표 헤더
- `12px`: 기본 본문과 주요 버튼
- `14px`: 판넬 부제
- `20px`: 판넬 제목
- `25px`: 화면 제목
- 수치에는 굵기 800 이상과 tabular 숫자를 사용한다.
- 한 화면에서 영문 눈썹은 하나의 크기와 색 역할만 사용한다.

## 컨트롤

- 아이콘 버튼 `38px`
- 기본 명령 `44px`
- 주요 명령 `48px`
- 모바일 핵심 터치 대상은 최종 화면 패스에서 최소 `44px` 확보
- hover는 표면 상승, active는 1px 눌림, focus는 이중 라임 링
- disabled는 색상 제거와 명도 감소를 함께 사용
- 색만으로 성공·실패·활성을 구분하지 않고 아이콘 또는 문구를 같이 표시

## 판넬과 모달

- 기본 판넬은 `--panel`, 선택·hover는 별도 표면 토큰 사용
- 카드 안에 장식용 판넬을 중첩하지 않는다.
- 모달은 동일한 backdrop, 외곽선, 헤더·본문·푸터 구조를 사용한다.
- 위험 모달만 빨강 경계를 허용한다.
- 스크롤바는 판넬과 같은 계열로 통일한다.

## 모션

- 일반 상태 전환 `120~180ms`
- 버튼 상태 변화는 크기 변형 대신 색·표면·1px 눌림 사용
- `prefers-reduced-motion`에서는 공통 전환 시간을 0으로 줄인다.
- 카드팩과 강화 FX는 별도 연출 규칙을 사용한다.
- 강화 성공 FX는 4단계·약 `3.12초`를 기준으로 하며, 고강화일수록 광륜과 입자·효과음 강도를 높인다.
- 성공 순간에도 카드 등급, 이름과 별 표식이 가려지지 않아야 한다.
- 모바일 가로에서는 `STAR UP` 표식을 카드 바깥 우측에 두고 안전 영역 안에서 축소한다.
