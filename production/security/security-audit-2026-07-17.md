# Security Audit Report

**Date**: 2026-07-17
**Scope**: 리뉴얼 클라이언트, 로컬 서비스, 서버 계약, 시즌1 이관 도구
**Release recommendation**: 서버 권위 어댑터 구현 전 공개 서비스 금지

## 요약

| 심각도 | 건수 | 상태 |
|---|---:|---|
| CRITICAL | 0 | 없음 |
| HIGH | 1 | PDB-9 서버 구현에서 필수 해결 |

| LOW | 1 | 추적 |

## HIGH

### SEC-001 로컬 상태가 아직 게임 결과의 권위자

- 위치: `src/renewal/local-game-service.js`, `src/renewal/storage.js`
- 현재 로컬 프로토타입은 개발자 도구로 포인트·카드·강화 결과를 바꿀 수 있다.
- 해결: PDB-5 계약대로 모든 지급·추첨·강화·전투 판정을 인증된 서버 트랜잭션으로 이동한다.
- 공개 배포 조건: `supabase-game-service` 적용, RLS, 서버 함수와 멱등성 저장 완료.

## MEDIUM

### SEC-002 외부 아이콘 CDN

- `index.html`이 고정 버전 Lucide를 `unpkg.com`에서 불러오지만 SRI가 없다.
- 해결: 정적 파일을 자체 호스팅하거나 배포 빌드에서 무결성 해시를 고정한다.

### SEC-003 CSP 미적용

- 동적 인라인 style과 외부 아이콘 스크립트 때문에 엄격한 CSP가 아직 없다.
- 해결: 자체 호스팅 전환 후 `script-src 'self'`, 제한된 `connect-src`, `object-src 'none'`을 배포 헤더에 적용한다.

## LOW

### SEC-004 카드 메타데이터 HTML 템플릿

- 카드명은 저장소의 빌드 산출물이라 현재 신뢰 경계 안에 있다.
- 외부 관리 화면에서 카드명을 받게 되면 출력 이스케이프와 입력 검증을 적용해야 한다.

## 완료 보완

- 클라이언트 번들에서 service role, 관리자·브릿지 비밀키 패턴 0건.
- 랭킹·월드보스 사용자 닉네임 `escapeHtml()` 적용.
- 로그인 키 원문 이관 금지. SHA-256 해시만 보존.
- `last_ip`는 시즌2 게임 상태로 이관하지 않는다.
- 명령 재전송은 같은 멱등성 키 사용, 다른 요청의 키 재사용 거부.
- 서버 오류·오프라인·revision 충돌에서 로컬 보상 확정 금지 계약 작성.

## 재감사 시점

PDB-9에서 실제 Supabase 함수, RLS와 서비스 어댑터 구현 직후 전체 재감사한다.
