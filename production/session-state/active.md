- `scripts/build-renewal-card-frames.py`는 S·SS·SSS 독립 마스터를 보존하고 F~A만 공통 형상에서 파생하도록 수정
- 프레임이 너무 얇다는 피드백 반영: 직선부 `3.4%`, 모서리 최대 `4.2%`로 상향하고 전 등급 사진 개구부를 런타임 `932x1332`로 재고정
- 강화 표기를 `+1~+9`에서 강화 별 아이콘 `×1~×9`와 사용자 용어 `1성~9성`으로 개편. 등급은 좌상단 검정 네온 코어로 유지하고, 강화는 검정 박스 없이 우상단 금속 별과 숫자만 띄우는 플로팅 마크로 통일
- 새 프레임 이미지를 다시 감싸던 CSS 등급색 외곽선과 내부 보조선 제거. 선택 상태 강조선만 유지
- 팩 결과의 `S·SS·SSS` 카드만 등급 영상 종료 후 뒷면에서 대기하고 사용자가 눌러 공개. `A` 이하 카드는 중간 공개에서 제외하고 최종 전체 결과창에서만 표시하며, 고등급이 없는 팩은 바로 최종 결과창으로 진행
- 도감을 시즌1 운영 방식처럼 사람별 섹션으로 재정렬하고 같은 사람 안에서는 높은 등급부터 배치. 미등록 카드는 목록·우측 미리보기·상세 모두 `assets/card-back.jpg` 뒷면으로 표시
- 로컬 호스트 전용 테스트 프로필 추가: 계정명 `MSTZ`, `1,000,000P`, 전체 207종 도감 등록 및 카드별 최소 1장 보유. 배포 호스트에는 적용되지 않음
- 카드 오픈 최종 결과창을 수량별 레이아웃으로 수정: 3~4장은 150px 중앙 정렬, 5~10장은 5열, 30~40장은 8열 스크롤. 이름용 광역 `span` 선택자가 등급·강화 마크를 카드 전체로 늘려 검게 덮던 충돌을 전용 클래스로 분리
- 미강화 카드의 불필요한 강화 별 `×0` 마크는 숨기고 `1성` 이상부터만 표시
- 팩 연출 건너뛰기를 버튼 대신 연출 화면 좌클릭으로 변경. 클릭 시 최종 결과로 닫지 않고 `S·SS·SSS` 수동 공개 화면으로 즉시 이동하며, 고등급이 2장 이상이면 `한번에 오픈하기` 버튼 제공
- `docs/RENEWAL-PRE-DATABASE-WORKLIST.md`를 DB 진입 전 단일 체크리스트로 추가
- 순서: UI·UX·브랜드 완성 → 무안내 플레이테스트 → 밸런스 잠금 → 상태 스키마 → 서비스 경계 → 서버 계약 → 오류 UI → 사전 검수 → 이관 명세 → DB 승인
